IaaS, PaaS y SaaS

Los tres modelos principales de servicio en la nube, según el nivel de control del usuario.

Comparativa

¿Quién gestiona qué?

Cuanto más alto el modelo, más gestiona el proveedor y menos el usuario.

Modelo 1

IaaS — Infraestructura como Servicio

El proveedor ofrece servidores virtuales, almacenamiento y redes. El usuario gestiona el sistema operativo y las aplicaciones.

¿Qué incluye?

Servidores virtuales, almacenamiento en disco, redes privadas y balanceadores de carga accesibles por API.

¿Para quién es?

Empresas que necesitan control total sobre la infraestructura sin comprar hardware físico.

Ejemplos

Amazon EC2, Amazon S3, Google Compute Engine, Microsoft Azure VMs.

Modelo 2

PaaS — Plataforma como Servicio

El proveedor ofrece un entorno de desarrollo completo. El desarrollador solo se enfoca en escribir el código de la aplicación.

¿Qué incluye?

Herramientas de desarrollo, bases de datos, servidores de aplicaciones y sistemas de despliegue automático.

¿Para quién es?

Desarrolladores y equipos de software que quieren crear aplicaciones sin administrar servidores.

Ejemplos

Google App Engine, Heroku, Microsoft Azure App Service, AWS Elastic Beanstalk.

Modelo 3

SaaS — Software como Servicio

El proveedor ofrece la aplicación lista para usar desde el navegador. El usuario solo necesita una cuenta.

¿Qué incluye?

Aplicación completa, actualizaciones automáticas, seguridad y almacenamiento gestionados por el proveedor.

¿Para quién es?

Cualquier usuario o empresa que quiera usar software sin instalación ni mantenimiento técnico.

Ejemplos

Gmail, Google Workspace, Microsoft Office 365, Zoom, Slack, Netflix, Dropbox.
